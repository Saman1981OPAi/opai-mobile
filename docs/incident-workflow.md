# Incident Workflow

## Purpose

The New Incident workflow provides a structured local draft surface for organizing incident details before future AI-assisted drafting and backend synchronization.

## Current Local Workflow

Supported local actions:

- Create a draft
- Edit a draft
- Save a draft
- Mark reviewed
- Archive
- Delete with confirmation
- Add persons involved
- Add witnesses
- Add notes
- Add attachment metadata
- Create follow-up reminders
- Create local calendar placeholders
- Prepare for AI review placeholder

## Incident Types

The local prototype includes incident type options such as Assault, Domestic Incident, Theft, Fraud, Mischief, Impaired Driving, Traffic Stop, Mental Health Call, Missing Person, Suspicious Person, Property Damage, Neighbour Dispute, and Other.

These are productivity categories only. OPAi does not provide legal classification, charge approval, or official report approval in this sprint.

## Required Warnings

New Incident drafts are local prototype records only and do not replace official police RMS, notebook requirements, reporting systems, supervision, policy, or legal obligations.

Do not enter real police records, confidential information, sensitive personal information, or real evidence into this testing version.
