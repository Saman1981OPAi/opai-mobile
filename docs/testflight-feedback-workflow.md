# TestFlight Feedback Workflow

## Intake Channels

- TestFlight screenshot feedback
- TestFlight crash feedback
- Email to support@opaiapp.com
- Sanitized GitHub issue for approved internal team members

## Required Fields

- Tester name or approved alias
- Device model and iOS version
- App build
- Screen/module and issue type
- Severity
- Steps to reproduce
- Expected and actual result
- Sanitized screenshot/video when useful
- Technical notes and current status

## Severity and Response

- Critical: crash, cannot launch, or cannot access the app; investigate the same day.
- High: major workflow blocked; investigate within 24 hours.
- Medium: workflow partly broken or confusing; plan for the next approved sprint.
- Low: typo, visual detail, or minor spacing; add to backlog.

Triage against `docs/beta-feedback-intake-system.md`. Remove all real police, evidentiary, personal,
medical, or confidential information before recording or sharing feedback. Feature requests are
deferred during the release freeze.
