# Apple Review Evidence Checklist

## Required Evidence

- [ ] App version, build number, source commit, and EAS build URL/identifier
- [ ] App Store Connect status and processing state
- [ ] Exact guideline and Apple message
- [ ] Reproduction steps on the same device class and iOS version when available
- [ ] Screenshot or screen recording containing demo data only
- [ ] Relevant configuration, permission, asset, or metadata comparison
- [ ] Public URL response and title when the issue concerns a URL
- [ ] Before/after evidence for the correction
- [ ] Validation results and regression scope
- [ ] Release-owner approval of the final response

## Evidence Safety

- Never attach passwords, authentication codes, signing material, API keys, Apple credentials, or
  private certificate data.
- Never attach real police records, evidence, incident details, statements, or personal data.
- Redact device/account identifiers that Apple does not need.
- Preserve original Apple wording; do not paraphrase it in the issue record.
- Store links to approved artifacts rather than copying sensitive files into Git.
