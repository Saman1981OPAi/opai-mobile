# Security And Privacy Notes

OPAi Police will handle sensitive workflows. The mobile app must be designed around privacy and consent from the start.

## Required Controls

- Encryption in transit.
- Encryption at rest.
- Secure authentication.
- Biometric login support.
- Role-based access control.
- Audit logs.
- Minimal sensitive-data storage.
- Consent for calendar, files, microphone, camera, notifications, and biometrics.

## Mobile UX Rules

- Do not show incident details in notification previews by default.
- Use generic critical reminders for lock-screen alerts.
- Make sensitive permissions contextual.
- AI drafts must require officer review.
- File upload, microphone, camera, and calendar states must be visible and reversible.

## AI Rules

- AI output is assistance, not authority.
- Report drafting must be reviewable.
- Legal and policy search must preserve source clarity when sources are available.
- Summaries should state which user notes or files were used.
