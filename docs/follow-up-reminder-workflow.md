# Follow-Up Reminder Workflow

## Purpose

The Follow-Up workflow lets users track local prototype reminders connected to reports, incidents, or operational tasks.

## Current Sprint 009 Behavior

Fields:

- Title
- Related incident ID
- Due date
- Priority: low, medium, high
- Notes
- Reminder enabled
- Reminder lead time
- Status: open, completed, overdue

Supported local actions:

- Add
- Edit
- Delete
- Complete/reopen
- Schedule or cancel local reminders

## Safety Notice

Follow-up reminders are local prototype data only. Users should not enter real police records, evidence, confidential information, or sensitive personal information.
