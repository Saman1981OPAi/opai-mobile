# Internal TestFlight Group Setup

## Group

- Name: OPAi Internal Testers
- Scope: approved internal iOS beta reviewers only
- Build: target `0.1.0 (21)` after processing

## Suggested Coverage

- Founder/Admin
- Technical reviewer
- UI/UX and accessibility reviewer
- Legal/privacy reviewer
- Police workflow reviewer using demo scenarios only
- General usability tester

## Setup

1. In App Store Connect, open OPAi Police -> TestFlight -> Internal Testing.
2. Create or select `OPAi Internal Testers`.
3. Add only authorized App Store Connect users eligible for internal testing.
4. Assign the processed build and complete any missing beta/compliance fields.
5. Paste the tester note below and send invitations.
6. Remove access promptly when a tester leaves the approved group.
7. Collect feedback through TestFlight or support@opaiapp.com and track sanitized reports.

## Invitation Note

Please use demo data only. Do not enter real police records, confidential information, sensitive
personal information, real evidence, real statements, or official documents.

Internal testers must not rely on mock AI, translation, reminders, or incident workflows for
official, legal, medical, emergency, evidentiary, or operational decisions.
